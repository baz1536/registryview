Garbage Collection — reclaim disk space after deleting tags ▼

Deleting tags via RegistryView removes the manifest references, but the underlying image layers remain on disk until garbage collection is run on the registry host.

⚠️ Warning: The paths below are specific to your environment. Using incorrect paths could target the wrong data. Double-check the volume name or path matches your registry setup before running.

Stop the registry first, run garbage collection using a temporary container against the same data volume, then restart:

registry:3

docker stop registry

docker run --rm \
  -v registry_data:/var/lib/registry \
  registry:3 garbage-collect \
  /etc/distribution/config.yml

docker start registry

registry:2

docker stop registry

docker run --rm \
  -v registry_data:/var/lib/registry \
  registry:2 garbage-collect \
  /etc/docker/registry/config.yml

docker start registry

Replace registry_data with your volume name or the host path to your registry's data directory. The registry container must be stopped before running garbage collection.

Add --delete-untagged before the config path to also remove layers belonging to untagged images. Run with --dry-run first to preview what will be deleted.
