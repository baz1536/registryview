RegistryView

Browse your private Docker registries

RegistryView is a self-hosted web UI for browsing private Docker registries. View repositories, inspect tags, and delete images — all from a clean, secure interface.

🗂️

Repositories

Browse all images in your connected registries. Expand any repository to inspect its tags, image digests, and layer sizes.

🔒

Registries

Connect multiple private Docker registries. Credentials are stored encrypted at rest using AES-256-GCM with a per-installation random salt.

🗑️

Delete Tags

Select one or more tags and delete them via the registry API. Deletion must be enabled on the registry host. Run garbage collection afterwards to reclaim disk space.

🧹

Garbage Collection

Deleting tags removes manifest references but not the underlying layers. Stop the registry, run garbage-collect with a temporary container, then restart to reclaim disk space.

Garbage Collection

registry:latest (v3) — reclaim disk space after deleting tags ▼

Stop the registry, run garbage collection using a temporary container against the same data volume, then restart. Replace /path/to/registry/data with your volume name or host path.

docker stop registry

docker run --rm \
  -v /path/to/registry/data:/var/lib/registry \
  registry:latest garbage-collect \
  --delete-untagged \
  /etc/distribution/config.yml

docker start registry

Run with --dry-run added before the config path to preview what will be deleted without making any changes.

registry:2 — legacy registry garbage collection ▼

registry:2 uses a different entrypoint and config path. The --delete-untagged flag is not supported on v2.

docker stop registry

docker run --rm \
  -v /path/to/registry/data:/var/lib/registry \
  registry:2 \
  garbage-collect \
  /etc/docker/registry/config.yml

docker start registry
