Garbage Collection — reclaim disk space after deleting tags ▼

Deleting tags via RegistryView removes the manifest references, but the underlying image layers remain on disk until garbage collection is run on the registry host.

⚠️ Warning: The paths below are specific to your environment. Using incorrect paths could target the wrong data. Double-check the volume path matches your registry setup before running.

Stop the registry first, run garbage collection using a temporary container against the same data volume, then restart:

docker stop registry

docker run --rm \
  -v /opt/docker-volumes/registry/data:/var/lib/registry \
  registry:2 garbage-collect \
  /etc/docker/registry/config.yml

docker start registry

Replace /opt/docker-volumes/registry/data with the path to your registry's data volume. The registry container must be stopped before running garbage collection.
